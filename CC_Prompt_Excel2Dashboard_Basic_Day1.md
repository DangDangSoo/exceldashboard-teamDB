# [Day 1] Excel2Dashboard(Basic) — 뼈대 · 업로드 · 해석 · 기술통계

## 배경
`@PRD_Excel2Dashboard_Basic_v0.1.md`의 Basic 버전을 만든다. 이번 Day 1은 **첫 뼈대**다. 파일을 올려 표로 보여주고, 각 열의 타입을 판별하고, 기술통계를 내는 데까지. **차트·집계·저장은 이번에 만들지 않는다.**

- 스택: **Python + FastAPI + pandas**. 프론트는 가벼운 HTML/JS(프레임워크·빌드도구 없이).
- 저장 없음: 업로드 파일은 서버 임시 세션에만. 새로고침하면 사라져도 된다(Basic 전제).
- 참고: PRD 2절(아키텍처·이음새), 3.1~3.3절(업로드·타입판별·기술통계).

---

## 0. 프로젝트 뼈대 (이음새 3원칙을 처음부터 지킬 것)
- 폴더 구조 예시:
  - `app/main.py` — FastAPI 라우터(호출만)
  - `app/core/` — **순수 로직 모듈**: `parse.py`, `infer_types.py`, `stats.py`
  - `app/models.py` — JSON 데이터 계약(dataset, column meta)
  - `static/` 또는 `templates/` — 업로드 화면·미리보기·통계 표시용 프론트
- **이음새 원칙(PRD 2.1):**
  1. 로직(parse/infer/stats)은 라우터·UI와 분리된 **입력→출력 순수 함수**로.
  2. 데이터셋을 **직렬화 가능한 JSON**으로 표현(아래 1번 스키마).
  3. 데이터셋에 **`uuid`** 부여.

## 1. 데이터 계약(JSON 스키마) 먼저 정의
- 파싱 결과를 담는 `Dataset` 형태를 코드로 고정한다. 최소 필드:
  - `id`(uuid), `filename`, `row_count`, `col_count`
  - `columns`: 각 열마다 `{ name, dtype(numeric|categorical|datetime|boolean), missing_rate }`
  - `preview`: 상위 N행(기본 20행)
- 이 JSON이 이후 Pro에서 저장·재호출되는 단위임을 주석으로 명시.

## 2. 업로드 · 파싱
- `xlsx`, `xls`, `csv` 업로드 엔드포인트.
- **한 파일 = 한 탭**, **1행 헤더 · 2행부터 데이터** 전제.
- CSV 인코딩(UTF-8 / EUC-KR) 자동 감지 시도.
- 파일 크기 상한 **10MB / 5만 행**. 초과 시 4xx + 명확한 메시지.
- 파싱 성공 시 위 `Dataset` JSON을 반환.

## 3. 열 타입 자동 판별 (+ 수동 교정)
- 각 열을 **numeric / categorical / datetime / boolean** 중 하나로 분류하고 **결측률** 계산.
  - 날짜가 텍스트로 들어온 경우 datetime 인식 시도.
- 프론트에서 열마다 타입을 **드롭다운으로 수동 교정** 가능. 교정 시 통계·표시를 다시 계산.

## 4. 표 미리보기 · 기술통계 대시보드
- **미리보기:** 상위 20행 표 + 전체 행/열 수 표시.
- **기술통계:**
  - 수치열: 개수, 결측 수, 평균, 중앙값, 최소/최대, 표준편차, Q1/Q3.
  - 범주열: 고유값 수, 최빈값, 상위 카테고리 빈도.
- 화면 구성 순서: **요약(행·열·결측 요약) → 통계표**. (차트 자리는 Day 2에서 채운다.)

## 5. 오류 처리 (Day 1 최소선)
- 잘못된 파일·형식·손상 파일 → 죽지 말고 4xx + 사용자에게 **보이는 알림**.
- 빈 파일·헤더만 있는 파일 → 명확한 안내.
- 서버 예외로 프로세스가 죽지 않게.

## 제약
- Python + FastAPI + pandas만. 프론트는 바닐라 HTML/JS.
- 저장소·DB·localStorage 금지(Basic은 세션 1회성).
- 이음새 3원칙(모듈 분리 / JSON 계약 / id)을 반드시 지킬 것.

## 완료 기준 (구현 후 이 순서로 점검)
1. **업로드:** xlsx·xls·csv가 올라가고, 10MB/5만 행 초과·손상 파일은 명확히 거절된다.
2. **파싱·미리보기:** 상위 20행 표와 전체 행/열 수가 정확히 표시된다.
3. **타입 판별:** 열 타입이 자동 분류되고 결측률이 맞으며, 드롭다운 수동 교정이 통계에 반영된다.
4. **기술통계:** 수치·범주 통계가 pandas 계산과 일치한다(값 한두 개 직접 검증).
5. **견고성:** 잘못된 입력·빈 파일에도 서버가 죽지 않고 보이는 알림으로 처리된다.
6. **이음새:** parse/infer/stats가 순수 모듈로 분리돼 있고, Dataset이 JSON·uuid로 표현된다.

## 진행 방식
- 먼저 **폴더 구조와 Dataset JSON 스키마**를 짧게 제시한 뒤 시작.
- 순서: 스키마 정의 → 파싱 → 타입판별 → 통계 → 미리보기 UI → 오류 처리.
- 끝나면 위 6개 점검 절차를 안내하고, **샘플 엑셀 1개로 실제 업로드 데모**까지 보여줄 것.
