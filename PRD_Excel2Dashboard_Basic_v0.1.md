# Excel2Dashboard — Basic 버전 PRD v0.1

## 1. 배경 / 위치

- **3부작 중 1부.** Basic(이번) → Pro(영속성) → Team(다중 사용자 협업). 각 단계는 지난 것을 버리지 않고 한 겹만 얹는다.
- **목표 사용자:** 보고서 작성이 필요한 사무직 중간 관리자. 코딩·통계 전문가가 아니며, "엑셀을 올리면 알아서 보기 좋게 정리해 주는" 경험을 원한다.
- **Basic 한 줄 정의:** 엑셀/CSV를 업로드하면 자동으로 대시보드를 그려주고, 원하는 분석 결과를 PNG로 내려받는다.
- **Basic이 하지 않는 것:** 저장·프로젝트·재호출·다중 사용자. Basic은 **세션 내 1회성** — 새로고침하면 데이터가 사라진다. (저장은 Pro, 협업은 Team의 몫.)

---

## 2. 스택 / 아키텍처

- **백엔드:** Python + FastAPI + pandas. 파싱 · 타입 판별 · 통계 · 집계 · 차트 생성을 담당하는 분석 서버.
- **차트 렌더링:** 서버에서 **matplotlib으로 PNG 생성** (내보내기 형식이 PNG only이므로 화면 표시와 다운로드를 같은 이미지로 통일 — 일관성·단순성 확보).
- **프론트:** 가벼운 HTML/JS. 파일 업로드, 대시보드 렌더, 차트별 PNG 다운로드 버튼. (프레임워크·빌드도구 최소화, 강의 가독성 우선.)
- **저장:** 없음. 업로드 파일은 서버의 임시 세션 메모리/임시경로에만 두고, 세션 종료·새로고침 시 폐기.

### 2.1 3버전 확장을 위한 "이음새" (Basic에서 반드시 지킬 것)

1. **로직 ↔ UI/라우팅 분리.** `parse`, `infer_types`, `stats`, `aggregate`, `make_chart`를 입력→출력의 **순수 함수 모듈**로 둔다. FastAPI 라우터는 이 모듈을 호출만 한다.
2. **직렬화 가능한 JSON 데이터 계약.** "데이터셋(dataset)"과 "분석/차트 설정(analysis config)"을 처음부터 JSON으로 표현한다. Basic에선 메모리에만 있지만, 이 JSON이 Pro에서 DB에 저장되고 Team에서 공유되는 **단위**가 된다.
3. **id 부여.** 데이터셋·분석 결과에 지금부터 `uuid`를 붙인다. "전역 상태 하나에 다 담기" 식 구조를 피한다.

> 이 세 가지만 지키면 Pro는 "저장 계층만 얹기", Team은 "계정·공유만 얹기"가 되어 재작성이 발생하지 않는다.

## 3. 기능 요구사항

### 3.1 업로드 · 파싱
- `xlsx`, `xls`, `csv` 지원. **한 파일 = 한 탭** 전제. **1행 = 단일 헤더, 2행부터 데이터** 전제.
- CSV 인코딩(UTF-8 / EUC-KR) 자동 감지 시도.
- 파싱 후: 상위 N행 **표 미리보기** + 전체 행/열 수 표시.

### 3.2 열 타입 자동 판별
- 각 열을 **수치 / 범주 / 날짜 / 불리언**으로 분류하고 **결측률**을 표시.
- 오판 대비 **사용자 수동 교정**(드롭다운으로 타입 변경) 지원.

### 3.3 기술통계 대시보드
- **수치열:** 개수, 결측 수, 평균, 중앙값, 최소/최대, 표준편차, 사분위(Q1/Q3).
- **범주열:** 고유값 수, 최빈값, 상위 카테고리 빈도.

### 3.4 시각화 (자동 추천 + 수동 선택)
- 지원 차트: **히스토그램**(수치 분포), **막대**(범주 빈도), **꺾은선**(날짜×수치 추세), **산점도**(수치×수치), **상관 히트맵**(수치열 전체).
- 열 타입에 맞춰 적절한 차트를 **자동 추천**하고, 사용자가 축·열을 바꿔 **수동 생성**도 가능.

### 3.5 그룹 집계 / 피벗 *(확장 범위)*
- **그룹 집계:** 범주열 기준 group-by + 집계함수(합 · 평균 · 개수 · 최소 · 최대) → 결과 표 + 막대 차트.
- **간단 피벗:** 행=범주, 열=범주, 값=수치(집계함수 적용) 형태의 피벗 표.

### 3.6 이미지 다운로드
- 각 차트를 **PNG**로 다운로드. 파일명에 차트 종류·열 이름 반영(예: `histogram_매출액.png`).

## 4. 비기능 / 제약

- **파일 크기 상한**을 정하고(예: N MB 또는 M행) 초과 시 명확히 안내.
- **지저분한 데이터 방어:** 빈 행, 타입 혼재, 날짜가 텍스트로 들어온 경우 등에서 **죽지 말고 경고**로 처리.
- **오류 처리:** 조용히 실패 금지. 파싱 실패·잘못된 입력은 **사용자에게 보이는 알림**으로.
- **스택 제약:** Python + FastAPI + pandas + matplotlib. 프론트는 바닐라 지향, localStorage 금지(Basic은 저장 자체가 범위 밖).

## 5. 만들지 말 것 (Basic 범위 밖)

- 저장 · 프로젝트 단위 · 엑셀 재호출 · 분석결과 저장  → **Pro**
- 다중 사용자 · 계정 · 로그인 · 공유 · 권한  → **Team**
- 다중 시트, SVG 내보내기, 실시간 협업, 예측/머신러닝

---

## 6. 단계 분할 (실습 진행)

- **Day 1 — 뼈대·해석:** 업로드 → 파싱 → 열 타입 자동 판별(+수동 교정) → 표 미리보기 → 기술통계. *(만들지 말 것: 차트·집계.)*
- **Day 2 — 시각화·내보내기:** 차트 자동 추천 + 수동 생성(히스토그램/막대/꺾은선/산점도/상관 히트맵) + PNG 다운로드. *(만들지 말 것: 그룹집계·피벗.)*
- **Day 3 — 집계·다듬기:** 그룹 집계 + 간단 피벗 + 견고화(오류·큰 파일·지저분한 데이터) + README·자체평가.

> 호흡이 급하면 Day 2를 "기본 차트 / 상관·산점도"로, Day 3를 "집계·피벗 / 견고화·문서"로 반쪽 내어 4~5단계로 확장 가능.

---

## 7. 완료 기준 (구현 후 이 순서로 점검)

1. **업로드·파싱:** xlsx/xls/csv가 브라우저에서 올라가고, 미리보기·행열 수가 정확히 표시된다.
2. **타입·통계:** 열 타입이 자동 판별되고 수동 교정이 되며, 기술통계 수치가 pandas 계산과 일치한다.
3. **시각화:** 추천·수동 차트가 정상 렌더되고, 각 차트가 PNG로 정확히 다운로드된다.
4. **집계·피벗:** 그룹 집계·피벗 결과가 표와 차트로 맞게 나온다.
5. **견고화:** 지저분한 데이터·큰 파일·잘못된 입력에도 서버가 죽지 않고 명확히 안내한다.
6. **문서:** `README.md`(실행법·기능·구조)와 자체평가 리포트 존재.

---

## 8. 자체평가 축 (`EVALUATION.md` 초안용)

- **데이터 해석 정확성** — 타입 판별·통계 수치가 원본과 일치하는가.
- **시각화 적절성** — 자동 추천이 열 성격에 맞는가, 차트가 읽기 쉬운가.
- **내보내기 품질** — PNG가 화면과 동일하고 파일명이 유의미한가.
- **견고성** — 현실의 지저분한 엑셀에서 안 죽는가.
- **확장 준비도** — 이음새 3원칙(모듈 분리 / JSON 계약 / id)이 지켜졌는가 = Pro로 갈 준비.

판정 루브릭: 🟢 입증 / 🟡 부분 / 🔴 미달. **최종 판정은 사용자 확인 몫.**

## 9. 열어둔 결정 (다음에 정하면 좋을 것)

- 파일 크기/행 수 상한 구체값.
- 미리보기 표시 행 수(N).
- 대시보드 레이아웃(상단 요약 카드 → 통계표 → 차트 그리드 순서 등).
- 상관 히트맵에 포함할 수치열 상한(열이 너무 많을 때).
