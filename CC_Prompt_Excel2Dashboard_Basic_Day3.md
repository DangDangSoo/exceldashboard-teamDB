# [Day 3] Excel2Dashboard(Basic) — 그룹집계 · 피벗 + 견고화 + 문서 · 자체평가 (Basic v1.0)

## 배경
Day 1~2로 업로드 · 타입판별 · 기술통계 · 5종 차트 · PNG 다운로드가 동작한다. 이번 Day 3이 **Basic의 마지막 단계**로, (a) 마지막 분석 기능인 **그룹집계·피벗**을 얹고, (b) **견고화·다듬기**, (c) **문서화·자체평가**로 v1.0을 마무리한다. 새 기능은 그룹집계·피벗뿐이고 나머지는 정리·검증이다.

- Day 1~2 기능 **회귀 없이**.
- 저장·프로젝트·다중사용자는 여전히 범위 밖(→ Pro / Team).
- 참고: `@PRD_Excel2Dashboard_Basic_v0.1.md` 3.5절(집계·피벗), 7·8절(완료기준·평가축).

## 0. Day 1~2 재확인 (건너뛰어도 되면 건너뛰기)
- `Dataset` JSON, 차트 스펙 JSON, 5종 차트 렌더·PNG 다운로드가 그대로 살아 있어야 함.
- 집계 로직도 이음새 원칙대로: `app/core/aggregate.py`를 **순수 함수 모듈**로.

## 1. 그룹 집계 (`aggregate.py`)
- **group-by:** 범주열 1개(또는 그 이상) 기준으로 묶고, 수치열에 **집계함수(합 · 평균 · 개수 · 최소 · 최대)** 적용.
- 출력: **결과 표** + **막대 차트**(Day 2의 차트 모듈 재사용).
- 집계 설정도 JSON으로 표현(그룹열·값열·집계함수) — Pro의 "분석결과 저장" 단위와 일관.

## 2. 간단 피벗
- 행=범주열, 열=범주열, 값=수치열(집계함수 적용) 형태의 **피벗 표**.
- 조합이 너무 커지면(카테고리 폭발) 상한/안내로 방어.
- 피벗 결과도 PNG로 내려받을 수 있게(표 이미지 또는 히트맵 형태 중 택1, 화면=다운로드 일치).

## 3. 견고화 (Basic 전체 대상)
- **지저분한 데이터:** 빈 행·타입 혼재·날짜 텍스트·결측 많은 열에서도 **죽지 말고** 가능한 범위로 처리하거나 명확히 안내.
- **큰 파일:** 10MB / 5만 행 경계에서 정상 거절·안내. 경계 근처에서 응답이 멈추지 않도록.
- **일관된 오류 처리:** 전 엔드포인트에서 잘못된 입력·부적합 조합에 **일관된 4xx + 메시지**. 프런트는 요청 실패 시 **보이는 알림**(조용히 실패 금지).
- **서버 안정성:** 예외로 프로세스가 죽지 않게.

## 4. 반응형 · 접근성 · 다듬기
- 좁은 창에서 요약카드·통계표·차트 그리드·피벗표 레이아웃이 깨지지 않게.
- 버튼·드롭다운 키보드 접근, 텍스트 대비 확인.
- matplotlib 한글 폰트·라벨 깨짐 최종 점검.

## 만들지 말 것 (Basic 범위 밖 — 유지)
저장 · 프로젝트 단위 · 엑셀 재호출 · 분석결과 저장(→ Pro), 로그인 · 다중사용자 · 공유(→ Team), SVG 내보내기, 다중 시트, 예측/머신러닝, 인터랙티브 차트.

## 제약
- Python + FastAPI + pandas + matplotlib. 프론트는 바닐라 HTML/JS.
- 저장소·DB·localStorage 금지(Basic은 세션 1회성).
- 이음새 원칙 유지: `aggregate.py` 순수 모듈, 집계·피벗 설정 JSON 계약.
- Day 1~2 기능 회귀 금지.

## 완료 기준 (구현 후 이 순서로 점검)
1. **그룹 집계:** 범주열 기준 group-by + 집계함수 결과가 표·막대로 정확히 나온다(pandas와 값 일치).
2. **피벗:** 행·열·값 피벗 표가 맞게 나오고, 카테고리 폭발을 방어하며, PNG로 내려받아진다.
3. **견고화:** 지저분한 데이터·큰 파일·잘못된 입력·부적합 조합에도 서버가 죽지 않고 일관되게 안내한다.
4. **반응형·접근성:** 좁은 창·키보드 이동·한글 라벨에 이상 없다.
5. **문서:** `README.md`·`EVALUATION.md`가 존재하고, 이음새·Basic 범위·제외 항목이 명시돼 있다.
6. **전체 회귀:** Day 1~2 기능(업로드·타입·통계·5종 차트·PNG) 전부 정상.

## 진행 방식
- 먼저 **집계·피벗 설정 JSON 계약과 바꿀/추가할 파일**을 짧게 제시한 뒤 시작.
- 순서: 집계·피벗 → 견고화 → 반응형·접근성 → 문서·평가.
- 끝나면 위 6개 점검 절차를 안내하고, **샘플 엑셀로 업로드→통계→차트→집계/피벗→PNG까지 전체 흐름 데모**를 보여줄 것.
- 마지막에 **Basic v1.0 마무리 요약**과 함께, Pro(영속성)로 넘어갈 때 이음새 3원칙이 어떻게 재사용되는지 한 문단으로 정리할 것.
