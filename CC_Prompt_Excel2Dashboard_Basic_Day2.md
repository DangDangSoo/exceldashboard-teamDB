# [Day 2] Excel2Dashboard(Basic) — 시각화 · PNG 다운로드

## 배경
Day 1로 업로드 · 파싱 · 타입판별 · 기술통계가 동작한다. 이번 Day 2는 그 위에 **시각화 계층**을 얹는다. 열 타입에 맞는 차트를 자동 추천하고, 사용자가 직접 축을 골라 만들 수 있게 하고, 각 차트를 **PNG로 다운로드**한다. **집계·피벗·저장은 이번에 만들지 않는다.**

- 차트는 **서버에서 matplotlib으로 PNG 생성**한다. 화면에 보이는 이미지와 다운로드되는 파일이 **동일한 PNG**가 되도록(형식·일관성 확보).
- Day 1 기능 **회귀 없이**.
- 참고: `@PRD_Excel2Dashboard_Basic_v0.1.md` 3.4절(시각화), 3.6절(이미지 다운로드).

---

## 0. Day 1 재확인 (건너뛰어도 되면 건너뛰기)
- Day 1의 `Dataset` JSON(열 타입·미리보기·통계)이 그대로 살아 있어야 한다.
- 차트 생성 로직도 이음새 원칙을 따른다: `app/core/charts.py`를 **순수 함수 모듈**로 두고, 라우터는 호출만.

## 1. 차트 생성 모듈 (`app/core/charts.py`)
- 입력: `Dataset` + **차트 스펙(JSON)**, 출력: PNG 바이트.
- **차트 스펙(JSON) 계약**을 정의한다. 최소 필드:
  - `chart_type`(histogram | bar | line | scatter | heatmap)
  - `x`(열 이름), `y`(열 이름, 필요 시), `title`(선택)
- 이 스펙 JSON이 Pro에서 "분석결과 저장"의 단위가 됨을 주석으로 명시.

## 2. 지원 차트 (5종)
- **히스토그램:** 수치열 1개의 분포.
- **막대:** 범주열의 빈도(값별 개수). 카테고리가 너무 많으면 상위 N개 + "기타".
- **꺾은선:** x=날짜열, y=수치열의 추세.
- **산점도:** 수치열 × 수치열.
- **상관 히트맵:** 수치열 전체의 상관행렬. 수치열이 많으면 **상위 15열**(분산 큰 순)만.

## 3. 차트 자동 추천
- Day 1의 열 타입을 근거로 화면에 **추천 차트 세트**를 자동 제시:
  - 수치열마다 히스토그램, 범주열마다 막대, (날짜열+수치열) 있으면 꺾은선, 수치열 2개 이상이면 상관 히트맵.
- 사용자는 추천 카드를 눌러 바로 렌더.

## 4. 수동 차트 생성
- 사용자가 **차트 종류 + 축(열)**을 골라 만드는 UI.
- 선택한 열 타입이 차트에 안 맞으면(예: 산점도에 범주열) **막고 안내**.

## 5. 대시보드 레이아웃
- 화면 구성: **요약 카드 → 통계표(Day 1) → 차트 그리드**.
- 차트 그리드는 추천 차트 + 사용자가 만든 차트를 카드로 나열. 각 카드에 **[PNG 다운로드]** 버튼.

## 6. PNG 다운로드
- 각 차트를 PNG로 저장. **파일명에 차트 종류·열 이름 반영**(예: `histogram_매출액.png`, `scatter_광고비_매출액.png`).
- 화면 표시 이미지와 다운로드 파일이 동일할 것.

## 7. 오류 처리
- 차트에 맞지 않는 열 조합 → 렌더 대신 **보이는 안내**.
- 결측·비수치 혼재 열 → 죽지 말고 가능한 범위로 그리거나 명확히 안내.
- 한글 라벨 깨짐 방지(matplotlib 한글 폰트 설정).

## 제약
- Python + FastAPI + pandas + **matplotlib**. 프론트는 바닐라 HTML/JS.
- 저장소·DB·localStorage 금지(Basic은 세션 1회성).
- 이음새 원칙 유지: `charts.py` 순수 모듈, 차트 스펙 JSON 계약.
- Day 1 기능 회귀 금지.

## 완료 기준 (구현 후 이 순서로 점검)
1. **자동 추천:** 업로드 데이터의 열 타입에 맞는 추천 차트 세트가 뜬다.
2. **5종 렌더:** 히스토그램·막대·꺾은선·산점도·상관 히트맵이 모두 정상 렌더된다(한글 라벨 포함).
3. **수동 생성:** 차트 종류·축을 골라 만들 수 있고, 부적합 조합은 막고 안내한다.
4. **PNG 다운로드:** 각 차트가 유의미한 파일명의 PNG로 내려받아지고, 화면 이미지와 동일하다.
5. **견고성:** 부적합 조합·결측·많은 카테고리에도 서버가 죽지 않고 처리된다.
6. **회귀:** Day 1의 업로드·타입·통계가 그대로 동작한다.

## 진행 방식
- 먼저 **차트 스펙 JSON 계약과 바꿀/추가할 파일**을 짧게 제시한 뒤 시작.
- 순서: 차트 스펙 정의 → `charts.py` 5종 → 자동 추천 → 수동 생성 UI → PNG 다운로드 → 오류 처리.
- 끝나면 위 6개 점검 절차를 안내하고, **샘플 엑셀로 5종 차트 렌더 + PNG 다운로드 데모**까지 보여줄 것.
